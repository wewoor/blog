一种轻量级的插件式、组件化的 WEB IDE UI 开发框架

说明书摘要

本发明提供了一种插件式、组件化的 Web IDE UI 开发框架。其特征在于：可通过声明式的 React UI 组件快速完成一个轻量化、高性能的 WEB IDE 的搭建，通过框架提供的 Workbench、Theme、Locals、Language、Keybinding、Setting、Layout、Extension 等服务，轻松的完成对已有 IDE 应用的扩展和自定义开发。另外框架内置相应的状态管理、事件总线等功能模块，可轻松的完成对 React 应用的交互和扩展。所以，通过该框架，可轻松完成一种可扩展式的 Web IDE 的搭建。

权利要求书

  1. 基于 React 声明式 的 IDE UI 组件库；其特征在于： 是根据现代 Desktop IDE UI 交互研发的基础 声明式 UI 组件库；

  2. 可自定义、持续迭代的扩展（Extension）服务模块； 扩展会包括名称、类型、状态、描述、执行程序等基本部分组成，Extension 服务会通过 Provider 接口接收开发者自定义的插件，并在整个框架 Ready 后自动加载插件，另外，用户可在运行时随时禁用/启用该自定义扩展。

  3. Workbench 扩展服务模块； 通过 Workbench 服务，在框架提供的接口的基础上，进行 React UI 的自定义，从而满足所需要的扩展需求。

  4. Theme 扩展服务模块； Theme 服务可通过加载一个简单的 JSON 配置文件，即可完成 IDE UI 的样式切换，另通过适配，可兼容市面上主流 IDE 的 Theme。

  5. Locals 扩展服务模块； Locals 服务通过一个简单的 JSON 扩展配置，就可在应用中完成本地化的切换。达到 本地化与本身程序的分离，从而提升其本地化的扩展性。

  6. Language 扩展服务模块； 通过 Language 服务，可以加载不同类型的文件，并支持在线高亮、编辑，自定义交互等功能。该扩展包括另名称、类型，执行程序，渲染方式等等部分。

  7. Keybinding 扩展服务模块； 通过 Keybinding 服务，我们可以在扩展程序中，完成制定的 keyboard、鼠标等事件的自定义，从而丰富我们的 IDE 交互。

  8. Setting 扩展服务模块； Setting 服务，提供了操作编辑用户信息、Theme、Extension、Layout 等等功能的接口，其通过一种 JSON 配置化或者可视化的方式，让我们可以轻松 修改和自定义 IDE。

  9. Layout 扩展服务模块； Layout 服务提供了框架内置布局和自定义布局的 API，通过该服务可以实时操作 IDE 基本布局。

  10. 数据缓存服务模块； 内置的客户端存储（Storage）服务，用来缓存当前 Workspace 的相关信息。

  11. 应用状态管理（State Management)模块； 该模块提供了获取、更新 IDE 服务、Workspace 等状态的API，另外可通过该模块可扩展自己的应用状态

  12. 事件总线模块； 该模块用于模块、组件直接的数据通信

  13. 基于数据驱动视图的的 React 渲染模块； 该模块可将 IDE 视图的状态绑定到 React 组件，并实时渲染

说明书

领域

本发明涉及 Web 技术领域，特别是基于Web技术的 IDE 开发框架领域。

背景

随着 浏览器、Web、云技术的发展，越来越多的传统桌面（Desktop）端开发 IDE 工具开始云化，通过 Web 的形式给开发者提供在线编程的能力。例如微软的 Visual Studio Codespaces。Web 技术的优势就是，磨平了本地开发环境搭建的复杂性、本地计算能力、存储容量等等的局限性、让开发者打开浏览器即可马上进行在线可视化的开发、发布、运维等工作。所以拥有一款可扩展式的 IDE 框架，可轻松的完成此类应用的搭建和功能扩展。

发明内容

该发明提供了一种可快速搭建的、可自定义的、可扩展式的 Web IDE 框架。为了达到以上目标，具体实现如下：

其一，基于 Web 技术强大的渲染能力和 React.js UI组件开发库，我们研发了一套适用于 IDE 交互的组件库。另外自主研发的 React 应用框架，我们可以轻松的完成对视图的控制，并可提供对外操作视图的服务接口，从而达到 Workbench 可扩展的目的；其二，定义 Theme, Settings, Keybinding, Locals, Language, Layout 服务接口和和扩展程序的协议，开发者通过制定接口和扩展协议，可自定义开发满足框架要求的扩展程序，通过扩展（Extension）服务可动态加载、执行其扩展程序；其三，根据 一、二，我们提供了一套基本的 Workbench UI，开发者通过入口（Entry) 应用，可在自己的 Web 应用在快速引入默认的 IDE Workbench, 通过自定义扩展，从而提供基本的 IDE 开发功能；其四，如果框架默认的 Workbench 无法满足开发者需求，通过框架内置的组件库，服务程序可按自己的需求，重新组合各功能部件，完成 DIY。

其中，基础组件库的部分，依靠 React 组件化的技术，通过借鉴原桌面的的交互，研发相对应Web端的UI部件，这些基础组件可进行更高阶的自由组合（Compose），通过数据的变更，实现渲染内容的控制。例如 ContextMenu, Sidebar, Tree 等组件。

其中，数据和视图映射的部分，是通过定义可观察的（Observable）视图 Model，通过一个视图和Model的映射（Mapping）程序进行交互。当视图 Model 变更后，映射程序会自动更新对应的视图。

其中，扩展（Extension)服务对象，是向开发者提供组件，基础服务扩展的一个窗口。通过加载一个保含扩展定义的 Javascript 对象，扩展服务会待框架基础服务（WorkbenchService, SettingService 等）加载完毕后，当应用处于Ready阶段后，自动加载各扩展对象。加载过程中，扩展服务会根据其类型选择具体的扩展服务进行解析扩展程序，不符合规范的扩展程序，会通过异常的方式通知开发者，符合规范的扩展则会默认激活（active）执行。

其中，自定义 IDE Workbench 功能，则是通过该框架丰富的声明式组件，基础服务（React Mapping 程序，状态管理、事件模块等）对象，可自己 DIY 整个 IDE 交互界面。
